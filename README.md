# StickerMatch (iOS MVP)

A fan utility for completing a physical football sticker/card album. Track which
stickers you **have**, are **missing**, or have **repeated**, publish your spares +
needs to a marketplace, find **nearby** people whose album complements yours, and
arrange trades through **in-app chat** — no phone numbers shared.

> **Not an official app.** No FIFA / Panini / World Cup logos, player images, sticker
> artwork, or official album numbering/compilation are used. Player and country names are
> plain text only.
>
> **Safety:** _Meet only in public places. StickerMatch does not process payments, verify
> users, or guarantee trades._

No payments. Connections happen through in-app messaging; users can **block** and **report**.

> **Visibility model:** trade posts are public to **all signed-in users** (minus anyone you've
> blocked) and only while **active** — enforced server-side by RLS (`supabase/09_hardening2.sql`).
> "Nearby" is a **client-side** distance sort for convenience, _not_ a privacy boundary — so a
> post's meeting point/notes should never contain anything private.

## Tech stack

| Layer    | Choice                                                            |
|----------|-------------------------------------------------------------------|
| iOS app  | Swift, SwiftUI, MVVM, async/await                                 |
| Backend  | Supabase — Postgres + Auth + RLS + Realtime + Edge Functions      |

## 1. Set up Supabase

1. Create a project at <https://supabase.com>.
2. **SQL Editor** → run in order: `01` → `02` → `03` → `04` → `05` → `06` → `07` → `08`.
3. Verify the catalog: `select count(*) from public.stickers;` → **504**.
   (Replace it via Table Editor → `stickers` → Import CSV, shaped like `stickers_seed.csv`.)
4. **Deploy the edge function** (account deletion):
   ```bash
   supabase functions deploy delete-account
   ```
5. **Auth providers** (Authentication → Providers):
   - **Google** → enable; paste an OAuth Web client ID/secret (Google Cloud Console, with
     `https://<project>.supabase.co/auth/v1/callback` as the redirect URI).
     URL Configuration → Redirect URLs → add `stickermatch://login-callback`.
   - **Apple** → enable for release (needs paid Apple Developer setup; see checklist).
   - **Email** → **disable** (Authentication → Sign In / Providers → Email). The app is
     **OAuth-only** (no email/password UI), so the password sign-up endpoint is turned off to
     shrink the attack surface. Leaked-password protection is then **N/A** (no passwords exist).
6. Copy your **Project URL host** + **anon public key** (Project Settings → API).

## 2. Set up the iOS app

```bash
brew install xcodegen
xcodegen generate        # creates StickerMatch.xcodeproj (gitignored)
open StickerMatch.xcodeproj
```
`project.yml` pre-wires the `supabase-swift` package, Sign in with Apple capability, the
`stickermatch` URL scheme, the location usage string, the app icon, and the Info.plist ↔
xcconfig keys.

**Secrets:** edit `StickerMatch/Config/Secrets.xcconfig` (gitignored; a placeholder exists):
```
SUPABASE_HOST     = abcdwxyz.supabase.co        # host only, no https://
SUPABASE_ANON_KEY = <your anon public key>
```
> ⚠️ The app `fatalError`s on launch until these are real (by design).

Pick a Simulator/device, set your signing **Team** under Signing & Capabilities, and Run.
On first Google sign-in the DB trigger creates your profile automatically.

## How it works

- **Album** — browse by **team** with a progress hero and per-team progress rings; tap a
  team for its players. Real **flags**, global search, and a single **copies counter** per
  player (0 = missing, 1 = have, 2+ = repeated/spares). Status is derived from the count.
- **Marketplace** — posts near you, **sorted by distance** with a radius control
  (50/100/250 km / All); falls back to your **profile country** when location is off. Filter
  out your own posts; manage them under **My posts** (edit / swipe-to-delete). Each card has a
  **Message** button and the safety disclaimer.
- **Matches** — your album vs. nearby posts: *they have X you need* / *you have Y they need*,
  ranked by score, limited to your radius.
- **Messages** — realtime 1:1 chat (Supabase Realtime). Connecting never reveals a phone
  number; nicknames are snapshotted so profiles stay private.
- **Safety** — **block** (hides posts/matches/messages both ways, enforced in RLS) and
  **report** (reason + note) from any post, match, or chat; manage blocks in Profile.
- **Profile** — nickname, **country** (searchable picker) + **city** (MapKit autocomplete),
  meeting point, blocked users, sign out, and **real account deletion** (edge function wipes
  all your data via cascade).

### Privacy / data model
Your raw album (`user_stickers`) and profile are owner-private (RLS). Only what you publish in
a post is visible to others, and only to non-blocked users near you. Coordinates come from the
city you pick when posting; your live location is used on-device for sorting. `PrivacyInfo.xcprivacy`
declares the collected data (email, name, coarse location, user content).
